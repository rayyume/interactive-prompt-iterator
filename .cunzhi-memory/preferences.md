# 用户偏好设置

- 用户偏好：1) 需要生成总结性 Markdown 文档来梳理方案；2) 不需要生成测试脚本；3) 不需要自动编译，用户自己编译；4) 不需要自动运行，用户自己运行。选择方案 B（智能检测全部拒绝并自动重新生成）作为实现方案。
- 用户新需求：1) 希望平均交互只需一轮，表单一次性列出通用、深度、高级等多个维度的选项；2) 可以提供多个模式供用户选择；3) 参考 https://github.com/smkalami/prompt-decorators 的提示词 decoration 设计；4) 暂时不进行开发，先调研 GitHub 上的相关技术实现。
- 用户核心需求澄清：1) 装饰器只是控制回答风格的一部分；2) AI 需要根据用户的粗提示词自动分析并生成优化维度（不仅仅是预设选项）；3) 需要让用户直观看到哪些地方被优化了（可视化对比）；4) 需要调研更多 GitHub 项目，不能只看一个。
- ## 用户工作偏好

1. **文档生成**：完成任务后生成总结性 Markdown 文档
2. **测试脚本**：提供完整的测试脚本
3. **编译控制**：不要主动编译，用户自己编译
4. **自动测试**：完成修改后自动运行测试验证
